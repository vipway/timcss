# Taro / uni-app 接入边界与建议

本页的目标是避免口径失真，明确“可接入”与“已官方验证”的区别。

## 当前真实状态

- TimCSS 在扫描与原子生成层可用于 Taro / uni-app 项目。
- 仓库当前没有 `examples/taro-*`、`examples/uniapp-*` 的官方示例工程。
- 发布校验链路当前未包含 Taro / uni-app 的端到端 smoke 验证。

结论：当前可接入，但应按“项目内验证”使用，不应对外宣称“官方完整验证”。

## 推荐接入策略

1. 先把 `content` 指向框架实际模板文件（如 JSX、Vue SFC、WXML 等编译前模板）。
2. 先跑 `inspect`，确认关键原子类与变体被命中。
3. 再跑 `build`，确认目标端产物路径与体积符合预期。
4. 在业务项目 CI 内补一层 smoke，覆盖常用页面和状态类。

## 最小风险清单

- 避免过度依赖运行时动态拼接 class
- 对高价值页面做 candidate 快照回归
- 对安全区、吸底、发丝线等端侧差异点做真机验证

## 对外沟通建议

推荐写法：

- “已官方验证：H5、原生微信小程序”
- “Taro / uni-app：提供接入建议，欢迎项目共建验证”

不推荐写法：

- “Taro / uni-app 官方完整支持且无缝开箱”
